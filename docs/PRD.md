# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：平時看到喜歡的食譜經常散落在各個平台（瀏覽器書籤、手機截圖、備忘錄等等），導致真正要做菜時常常找不到。需要一個能夠集中管理的「食譜收藏夾」，方便隨手記錄與快速檢索。
- **目標用戶**：我自己（個人專屬的食譜管理與檢索工具）。
- **核心價值主張**：提供一個輕量、簡單且直覺的介面，快速統一儲存食譜圖文與分類，解決「今天吃什麼」的煩惱，以及「當初那個食譜存去哪了」的痛點。

## 2. 功能需求
以下為系統涵蓋的 5 大主要功能：

1. **新增食譜 (Add Recipe)**
   - *使用者故事*：作為使用者，我希望可以手動輸入食譜細節（名稱、所需食材、步驟、圖片及來源網址），以便將我在不同地方看到的食譜整合在同一個系統內。
2. **尋找食譜 (Search & Filter Recipes)**
   - *使用者故事*：作為使用者，我希望可以利用關鍵字搜尋食譜名稱或是食材，以便在做菜前快速找到想做的料理，不用一頁頁慢慢翻。
3. **食譜分類與標籤 (Categorization & Tagging)**
   - *使用者故事*：作為使用者，我希望可以替食譜加上分類或自訂標籤（例如：中式、甜點、快速料理、減脂、氣炸鍋），以便系統化地分類整理與篩選食譜。
4. **編輯與刪除食譜 (Edit & Delete Recipe)**
   - *使用者故事*：作為使用者，我希望可以隨時修改食譜內容（例如改良配方）或刪除不再需要的食譜，以便保持收藏夾內容的準確性與整潔。
5. **「今天吃什麼」隨機挑選 (Random Recipe Picker)**
   - *使用者故事*：作為使用者，我希望系統可以隨機從收藏中推薦一道食譜，以便解決我常常不知道下一餐要煮什麼的選擇困難。

## 3. 非功能需求
- **技術限制**：後端採用 Python 的 Flask 框架，前端使用 HTML/CSS 與 Jinja2 模板引擎進行頁面渲染，資料庫使用 SQLite 輕量化儲存資料。
- **效能需求**：系統於個人設備或本機運行，須保持輕量快速，頁面切換與搜尋的回應時間盡量低於 1 秒。
- **安全考量**：由於是個人使用的本地端服務，不強制實作複雜的會員註冊與權限管理（Auth），但需包含基礎防護（如防止 SQL Injection、XSS），並確保資料存取的穩定性。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**
  - 食譜的基本 CRUD 功能：新增、讀取（列表與明細頁）、更新、刪除。
  - 對食譜加上基本的分類標籤功能。
- **Should Have (應該有)**
  - 關鍵字搜尋功能。
  - 食譜圖片上傳與來源網址紀錄。
- **Nice to Have (加分項)**
  - 首頁的「隨機推薦」功能模組。
  - 資料庫/食譜的匯出與備份機制。
  - 食材熱量與營養素的自訂紀錄。
